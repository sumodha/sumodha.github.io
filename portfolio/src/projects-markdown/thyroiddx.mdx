# Introduction 
ThyroidDx is a full-stack web application that predicts **whether a patient has hypothyroidism, hyperthyroidism, or no thyroid disease based on routine blood work tests** (such as TSH, T3, TT4, T4U, FTI, TBG, etc), as well as other inidictions (such as if they are sick, if they have a tumor, etc). It **asks 17 questions** and **uses a neural network model for prediction**.

# Features
- **Diagnosis via Neural Network Model** 
  - Submits patient data to a trained ML model (via a Flask REST API) that returns a thyroid condition diagnosis in real time.
  - Model is pre-trained and served as a pickled .pkl file, making predictions fast and stateless.
- **Interactive Patient Form** - A guided input form collects 17 clinical features used by the model:
  - Demographics: age, and sex
  - Medical history (yes/no): currently on thyroxine (t4), on antithyroid medication, currently sick, pregnant, history of thyroid surgery, history of I-131 (radioactive iodine) treatment, self-reported query for hypothyroidism, self-reported query for hyperthyroidism, presence of goitre, and presence of tumor
  - Lab results (numeric): TSH (thyroid stimulating hormone), T3 (triiodothyronine), TT4 (total thyroxine), T4U (thyroxine uptake) and FTI (free thyroxine index)


# Data Processing
## PREPROCESSING DATA
- **Dropped features** that had over 50% null values.
- Autofilled hormone levels (for those not measured) using **imputation**.
- Used **one hot encoding** for target column. 
- Utilized **label encoding** for features with yes/no questions.
- **Standardized** numeric features (i.e lab results).
