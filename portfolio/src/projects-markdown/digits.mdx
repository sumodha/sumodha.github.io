# Introduction 
Digits is a full-stack web application that recreates the New York Times Digits game after its discontinutation. For a little bit more background on Digits, users are presented a target number that they have to reach using 6 numbers and 4 basic operations (addition, subtraction, multiplication, division). To reach the target number, you do not need to necessarily use all numbers given.

# Features 
- **Daily Puzzles** — Five new puzzles served each day, pre-seeded for the entire year.
- **Star Scoring** — Earn up to three stars per puzzle: exact match (☆☆☆), within 5 (☆☆), within 10 (☆).
- **Step-by-Step Undo** — Backtrack moves one at a time. Full operation history is displayed alongside the board.
- **Tab Persistence** — Switch freely between all five puzzles without losing progress (saved via local storage).
- **Integer-Only Division** — Division that produces a decimal is rejected, keeping the math clean.

# How to Play
1. **Pick two numbers** — Tap any two of the six number tiles. 
2. **Select an operation** — Choose +, −, ×, or ÷ from the toolbar. The result replaces the first number tile; the other is consumed.
3. **Repeat or submit** — Keep combining until a tile is within 10 of the target. Select it and hit submit.

## BACKEND
### API Reference 
```
GET /game 
Returns today's set of five puzzles. The server determines the current date internally, so no parameters required.

Response - 200 OK: 
{
  "items": {
    "_id": "...",
    "date": "4/22",
    "data": [
      {
        "target": 284,
        "numbers": [3, 7, 11, 14, 19, 22]
      },
      {
        "target": 156,
        "numbers": [2, 5, 9, 12, 18, 24]
      }
      // ... 5 puzzles total
    ]
  }
}

Response fields:
- items.date (string): Date key in M/D format, e.g. "4/22"
- items.data (array): Array of 5 puzzle objects
- items.data[].target (number): Target number the player tries to reach (1–900)
- items.data[].numbers (number[]): Six unique integers (1–25), sorted ascending
```
### Data Seeder 
A standalone Node.js script (insertData.js) **pre-generates all 365 daily entries and bulk-inserts them into MongoDB**. Target numbers are derived by repeatedly applying random valid operations to the seed numbers, guaranteeing each puzzle is solvable and the target stays within 1–900.
